## Purpose and Background

### Purpose of Agreement

This Data Sharing Agreement establishes the terms under which Provider will participate in the Santa Fe Street View By-Name List coordination system ("the System"), including:

1. What data Provider will contribute to the System
2. How Provider may access and use System data
3. Privacy protections and security requirements
4. Rights and responsibilities of both parties

### Background

The Santa Fe Street View By-Name List is a coordinated information system enabling homeless service providers to:

- Share appropriate information about individuals experiencing homelessness
- Coordinate services and avoid duplication
- Track progress toward housing stability
- Make data-informed decisions about resource allocation

System Operator, [FISCAL SPONSOR NAME], operates the System as legal custodian of shared data, ensuring HIPAA compliance and appropriate data governance.

## Individual Consent and Rights

### Informed Consent

**Provider Responsibility:**
Provider will obtain informed consent from individuals before adding their information to the System, using the standard consent form (Appendix B).

**Consent Form Elements:**

- What the System is and its purpose
- What information will be shared
- Who will have access to information
- How information will be used
- Individual's rights regarding their information
- How to request access, correction, or removal

**Documentation:**
Provider will maintain signed consent forms in individual's file.

**Exceptions:**
Emergency situations or street outreach may allow delayed consent with documentation of reason and follow-up consent obtained as soon as possible.

### Individual Rights

Provider agrees to support the following individual rights:

**Right to Access:**
Individuals may request to see their information in the System. Provider will facilitate requests and coordinate with System Operator.

**Right to Correction:**
Individuals may request correction of inaccurate information. Provider will review and make appropriate corrections within 10 business days.

**Right to Restrict:**
Individuals may request restrictions on which other providers can see their information. Provider will document restrictions and communicate to System Operator.

**Right to Removal:**
Individuals may request removal from the System. Provider will discuss implications and facilitate removal request if individual confirms. Historical aggregate data may be retained for continuity.

## Data Security and Protection

### System Operator Security Measures

System Operator maintains the following security safeguards:

**Technical Safeguards:**

- Encryption of data in transit (TLS/HTTPS) and at rest
- Secure authentication and password requirements
- Role-based access controls
- Audit logging of all access and changes
- Regular security monitoring and alerts
- Automated backups with encryption
- Business Associate Agreements with all technology vendors

**Organizational Safeguards:**

- HIPAA and data security training for all users
- Written security policies and procedures
- Incident response protocol
- Regular security audits
- Background checks for staff with system access

**Physical Safeguards:**

- Cloud-based system with enterprise security
- No local data storage on user devices
- Automatic session timeouts
- Secure disposal of any physical materials

### Provider Security Responsibilities

Provider agrees to:

**User Account Management:**

- ☐ Provide System Operator with list of authorized users and their roles
- ☐ Ensure all users complete required HIPAA and security training before access
- ☐ Notify System Operator within 24 hours when staff leave or change roles
- ☐ Review user access quarterly and confirm continuing need

**Device and Access Security:**

- ☐ Access System only from password-protected devices
- ☐ Never share login credentials
- ☐ Log out when leaving workstation unattended
- ☐ Use secure networks (no public WiFi without VPN)
- ☐ Report lost or stolen devices immediately

**Data Handling:**

- ☐ Do not download, copy, or screenshot System data without authorization
- ☐ Do not share information accessed in System except as permitted for service coordination
- ☐ Maintain physical security of any printed reports
- ☐ Shred or secure delete any physical copies when no longer needed

**Incident Reporting:**

- ☐ Report suspected security incidents immediately to System Operator
- ☐ Report any unauthorized access or disclosure
- ☐ Cooperate with incident investigation

## Permitted and Prohibited Uses

### Permitted Uses of Accessed Data

Provider may use information accessed in the System for:

✅ **Service Coordination:** Coordinating care and services with other providers  
✅ **Avoiding Duplication:** Knowing what services individual is already receiving  
✅ **Warm Handoffs:** Making appropriate referrals to other providers  
✅ **Case Planning:** Informing Provider's own case management for individuals served  
✅ **Provider Operations:** Internal planning and service delivery improvement  
✅ **Required Reporting:** De-identified aggregate reporting to funders (with System Operator approval)

### Prohibited Uses

Provider may NOT use information accessed in the System for:

❌ **Marketing:** Soliciting individuals for services they haven't requested  
❌ **Discrimination:** Denying services based on information in System  
❌ **Law Enforcement:** Sharing with police without consent or legal process  
❌ **Public Disclosure:** Publishing or posting identifiable information  
❌ **Employment Screening:** Using information for hiring or background checks  
❌ **Commercial Purposes:** Selling or sharing for non-service purposes  
❌ **Research:** Using for research without IRB approval and consent  
❌ **Personal Use:** Any use unrelated to professional service delivery

## Breach Notification and Incident Response

### Definitions

**Security Incident:** Attempted or successful unauthorized access, use, disclosure, modification, or destruction of information.

**Breach:** Unauthorized acquisition, access, use, or disclosure of PHI that compromises security or privacy.

### System Operator Obligations

If System Operator discovers a security incident or breach:

**Immediate Actions (within 24 hours):**

- Begin investigation and containment
- Notify Provider via phone and email
- Preserve evidence and logs

**Follow-up Actions (within 10 business days):**

- Complete investigation
- Provide written report to Provider including:
  - Nature and scope of incident
  - Individuals affected (if identifiable)
  - Actions taken to mitigate
  - Steps to prevent recurrence
- Assist Provider with individual notification if required
- Report to regulatory authorities as required by law

### Provider Obligations

If Provider discovers or suspects a security incident:

**Immediate Actions:**

- Report to System Operator immediately
  - Phone: [emergency number]
  - Email: [security email]
- Do not delete or modify any evidence
- Document what was observed

**Cooperation:**

- Provide relevant information for investigation
- Preserve evidence
- Assist with containment if within Provider systems
- Coordinate individual notification if required

### Individual Notification

If breach affects individuals' PHI:

- System Operator and Provider will collaborate on notification
- Notification within 60 days as required by HIPAA
- Provider responsible for notifying Provider's own clients
- System Operator responsible for notifying for System-level breaches

## Audit Rights

### Provider's Right to Audit

Provider has the right to:

- Request information about how Provider's shared data is being used
- Review security measures and compliance documentation
- Request audit logs showing access to Provider's clients' information
- Verify System Operator's HIPAA compliance

**Process:** Submit written request; System Operator will respond within 15 business days.

### System Operator's Right to Audit

System Operator has the right to:

- Review Provider's user activity logs
- Verify Provider compliance with security requirements
- Audit data quality and timeliness

**Process:** System Operator will provide 7-day advance notice except for security incidents.

### External Audits

System Operator will conduct independent security and HIPAA compliance audits annually:

- Results shared with Provider Advisory Committee
- Corrective actions implemented as needed
- Audit findings available upon Provider request

## Term, Termination, and Transition

### Term

This Agreement is effective as of the Effective Date and continues until:

- Either party provides written notice of termination, OR
- The System ceases operation

### Termination by Provider

Provider may terminate participation by providing 30 days written notice to System Operator.

**Upon Termination:**

- Provider access to System will be revoked
- Provider's contributed data will remain in System for continuity unless Provider requests deletion
- Historical aggregate data (de-identified) will be retained
- Provider will return or destroy any System materials or credentials

### Termination by System Operator

System Operator may terminate Provider's participation:

- For cause (material breach) with 10 days written notice and opportunity to cure
- If Provider fails to maintain required security standards
- If Provider violates prohibited uses of data

### System Discontinuation

If System operations cease:

- System Operator will provide 60 days notice if possible
- Data will be returned to contributing providers or securely destroyed
- Aggregate historical data may be retained by System Operator for continuity
